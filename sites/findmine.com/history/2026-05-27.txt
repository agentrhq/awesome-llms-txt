## What FindMine Does


FindMine turns a retailer's product catalog, styling guidance, and business constraints into automated, shoppable outputs, including:


- Shoppable outfits (Looks)
- Bundles and attachments (Complete the Look)
- Visually similar alternatives for discovery and out-of-stock fallback
- Dynamic lookbooks and themed edits for landing pages and email
- Cross-sell and upsell product recommendations
- Optional Creative outputs for ads and marketing where implemented


## Core Concepts (Glossary)


- Look / Outfit: A set of compatible items styled together
- Training Looks: Brand-provided examples that teach on-brand styling
- Brand POV / Style DNA: The brand's consistent aesthetic logic expressed in styling rules
- Business Rules / Guardrails: Hard constraints such as capsule rules or pricing restrictions
- Coverage: How much of the catalog is styled and how many Looks exist per product
- Substitution / Swap: Replacing unavailable items with brand-consistent alternatives

## How FindMine Works (High Level)


1. Ingest: Catalog data, inventory, pricing, and brand styling guidance (Training Looks + rules)
2. Decide: Multi-modal analysis of images and metadata, brand-specific styling logic, and guardrails. Optional session-based personalization without PII
3. Serve: APIs, widgets, and exports deliver Looks and content to channels; inventory-aware substitution keeps outputs shoppable
4. Learn: Performance analytics and merchandiser feedback improve decisions over time; human-in-the-loop controls support review and pinning


## API Information


FindMine provides a RESTful API that enables retailers to integrate our styling capabilities into their websites, apps, and other channels. Key API endpoints include:


### Complete the Look API
- Endpoint: `/api/v3/complete-the-look`
- Methods: GET, POST
- Purpose: Returns a list of products that complete an outfit based on a starting product
- Key parameters: product_id, product_color_id, customer_session_id
- Response: JSON with outfit recommendations


### Visually Similar API
- Endpoint: `/api/v3/visually-similar`
- Methods: GET, POST
- Purpose: Returns products that are visually similar to a reference product
- Key parameters: product_id, product_color_id
- Response: JSON with visually similar products ranked by similarity


### Item Details API
- Endpoint: `/api/v3/item-details`
- Method: POST
- Purpose: Update product inventory status to ensure recommendations only include available items
- Key parameters: product_id, product_in_stock, product_on_sale
- Response: Confirmation of update status


### Analytics API
- Endpoint: `/api/v3/analytics`
- Method: GET
- Purpose: Track user interactions with FindMine recommendations
- Key parameters: session_id, event_type, interaction
- Response: Tracking confirmation
